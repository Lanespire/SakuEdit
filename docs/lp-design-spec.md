## 1. 現行LPの問題点分析

### 1.1 構造上の問題

| 問題 | 具体例 | 影響 |
|------|--------|------|
| **ScrollRevealの過剰使用** | ナビロゴ、Hero全要素、Before/After、全機能カード、全ステップ等ほぼ全要素に適用 | 初回表示でコンテンツが空白。スクロールするたびに「まだ来てない」感がある。ユーザーはページの内容を事前に把握できず、情報の取得に遅延が生じる |
| **主役が不明確** | Heroに「ヘッドライン」「サブコピー」「CTA2つ」「3カード」「補足テキスト」が全て詰まっている | 視線が散り、3秒で何のサービスか伝わらない |
| **同じメッセージの反復** | 「80%を自動化」がHero・Pain Point・Featuresの3箇所に登場。「8時間→3分」もHero・Before/After・CTAで3回 | 情報密度が低く、スクロールしても新しい情報が得られない感覚 |
| **セクション間の論理断絶** | Hero → Before/After → Demo → Pain Point → Features → How it Works → CTA という順序 | Before/Afterの後にDemoが来て、その後にPain Pointが来る。「見せる→感じさせる→理解させる」の流れが崩壊 |
| **Before/After が数字だけ** | 「8時間」「3分」のテキストのみ | 具体的に何がどう変わるのかの視覚的証拠がない。信頼性が低い |

### 1.2 コピー上の問題

| 問題 | 具体例 | 改善方向 |
|------|--------|----------|
| **曖昧な価値提案** | 「新しい動画編集の形を手に入れよう」 | 何が「新しい」のか不明。具体的なベネフィットに置き換える |
| **ターゲットが見えない** | 誰に向けたメッセージか不明 | 「動画を作ってるけど編集が面倒な人」を明確にする |
| **機能説明が均一** | 6機能が同じサイズ・同じフォーマットで並ぶ | コア機能（無音カット・字幕・サムネ）と付加機能で階層化する |
| **信頼構築が完全に不在** | ロゴ/ユーザー数/テスティモニアル/第三者評価のいずれもなし | 全競合が3種以上の信頼要素を配置（Descript: 16社ロゴ、VEED: G2 4.9/12Mユーザー、Opus Clip: 16M+ユーザー） |
| **CTA表現の不統一** | 「今すぐ始める」「無料で始める」が混在 | UX調査: 「無料で試す」（Trial）系が「登録」系より104%効果的 |

### 1.3 アニメーション上の問題

| 問題 | 具体例 | 改善方向 |
|------|--------|----------|
| **意味のないアニメーション** | ロゴがleftからスライドイン | ロゴは最初から見えているべき |
| **ProcessingToEditorDemoのループ** | 12秒ごとにリセットされる | ユーザーがページに戻ったときに壊れた状態で表示される可能性 |
| **Staggerが遅い** | Pain Pointの3カードが0.15秒ずつ遅延 | 「ちょっと待つ」感覚が不快 |

### 1.4 削ぎ落とすべき要素

| 要素 | 理由 |
|------|------|
| Hero内の3カード（「無音カット」「字幕生成」「テンポ調整」） | Heroの主役を奪う。Features セクションで詳述すれば十分 |
| Hero内の補足テキスト（「ブログ記事の動画化・SNS投稿に最適」） | 主メッセージを薄める |
| Before/Afterの「Before/After」ラベル | 「8時間→3分」の対比が自明。ラベル不要 |
| Pain Pointセクション全体 | 既にBefore/Afterで痛みは示している。情報の重複 |
| How it Worksの「3ステップ」パターン | どのSaaSでも同じ。差別化にならない。Demoで代替可能 |
| FooterのSNSリンク（未設定のTwitter/Discord） | リンク切れは信頼性を下げる |

## 2. 情報アーキテクチャ

### 2.1 セクション構成と順序

新しい構成は「注目→証拠→理解→信頼→行動」のフローに基づく（UX調査: Z字パターン + 「Proof before persuasion」原則）。

```
Navigation（固定 + モバイルSticky CTA）
  └─ ロゴ + ナビリンク + CTA
  └─ モバイル: スクロール時にSticky CTAを表示（UX調査: +18%コンバージョン）

Section 1: Hero（注目 — 5秒で伝える）
  └─ ヘッドライン + サブヘッド + CTA + 画面キャプチャ
  └─ Z字パターン: 左上ロゴ → 右上CTA → 左下ヘッドライン → 右下メインCTA

Section 2: ソーシャルプルーフ（即座の信頼構築）
  └─ ロゴバー or ユーザー数表示
  └─ ※ローンチ初期は「ベータテスト中」「先行ユーザー募集中」でも可
  └─ UX調査: 信頼要素は機能説明の「前」に置く

Section 3: Before / After（具体的成果の証拠）
  └─ スプリットスクリーン: 従来の編集フロー vs SakuEditのフロー
  └─ テキスト数字だけでなく、ワークフローの視覚的対比

Section 4: 3つのコア機能（何が自動化されるか）
  └─ 無音カット / 字幕生成 / サムネイル生成
  └─ 各機能を大きめに、実画面に近いビジュアルで

Section 5: デモ（体験の構築）
  └─ ProcessingToEditorDemo（改良版）

Section 6: 付加機能（深掘り層向け）
  └─ チャット編集 / テンポ調整 / スタイル分析
  └─ 控えめなリスト形式

Section 7: 料金（ハードルを下げる）
  └─ Free プランを強調 + Pro への導線

Section 8: CTA（最終アクション）
  └─ シンプルなCTA + 「クレジットカード不要」+ 「90分無料」の具体数値

Footer
  └─ ロゴ + リーガルリンク
```

### 2.2 各セクションの主役要素と優先順位

| セクション | 主役 | 脇役 | 排除 |
|----------|------|------|------|
| **Hero** | ヘッドライン1行 | サブヘッド + CTA + 画面キャプチャ | 3カード、補足テキスト、デモボタン |
| **ソーシャルプルーフ** | ロゴ or ユーザー数 | — | テスティモニアル長文（ここでは不要） |
| **Before/After** | ワークフロー対比のビジュアル | 時間数値（8時間→3分） | 長いテキスト説明 |
| **コア機能** | 3つの機能カード（大きめ・ビジュアル重視） | 各機能の1行説明 | アイコン羅列、均等割り |
| **デモ** | ProcessingToEditorDemo | 「アップロードするだけ」の1行コピー | 過剰なアニメーション説明 |
| **付加機能** | 3つのサブ機能をリスト形式で | 簡潔な説明 | カード形式 |
| **料金** | Freeプランの「90分無料」 | Pro/Businessプラン | 4プラン全てを同等に扱うこと |
| **CTA** | 「無料で試す」ボタン | 「クレジットカード不要」+「90分無料」 | 過剰なコピー |

### 2.3 視線フローの設計（Z字パターン）

各セクション内でミニZ字パターンを繰り返す構造。UX調査によると、CTAはZ字の終点（右下）に配置するのが最も効果的。

```
Hero（ファーストビュー — 5秒ルール）
  ┌──────────────────────────────┐
  │ [ロゴ]          [ナビ] [CTA]│  ← Z字の起点
  │        ╲                    │
  │         ╲                   │
  │ [ヘッドライン]              │
  │ [サブヘッド]                │
  │ [画面キャプチャ]    [CTA]   │  ← Z字の終点
  └──────────────────────────────┘
  ↓ 5秒で「自分ごと化」完了

ソーシャルプルーフ
  ↓ ロゴ or 数値で「他の人も使ってる」→ 信頼
  ↓ UX原則: 「Proof before persuasion」（説得の前に証拠）

Before/After
  ↓ 従来フロー vs SakuEditフローの視覚的対比
  ↓ 「これなら楽だ」と直感で理解

コア機能
  ↓ 「無音カット」→「字幕生成」→「サムネイル」の順で
  ↓ 左→右（デスクトップ）/ 上→下（モバイル）

デモ
  ↓ 動いている画面を見て「本当に動くんだ」と確信

料金
  ↓ 「無料で使える」→安心→試してみよう

CTA
  ↓ 最終的な行動
```

## 3. コピーライティング

### 3.1 ヘッドライン案（3案）

#### 案A（推奨）: 完了形の驚き型
```
動画編集の80%は、
もう終わっている
```
**狙い**: 「80%」の具体性 + 完了形の驚き。「面倒な作業のほとんどが既に自動化されている」ことを暗示。UX調査によると、具体的な数字を含むヘッドラインは抽象的なものより27%以上効果的。Notionの「One workspace. Zero busywork.」のように短く成果で語る。

**サブヘッド**: アップロードしたその瞬間に。無音カット、字幕、サムネまで自動で。

#### 案B: 面倒さ共感型
```
動画編集の面倒な作業を
自動で片付ける
```
**狙い**: 「面倒」という感情に共感 + 「自動」という解決策を提示。AIという単語を使わず、結果にフォーカス。VideoPocket AIの「動画編集の面倒な作業を、AIが一瞬で片付けます」と同系統だがよりシンプル。

#### 案C: 時間短縮型
```
1日かかる編集を
3分で終わらせる
```
**狙い**: 数字の対比でインパクト。Opus Clipの「Create 10x faster」と同系統だが、日本語として自然な表現。損失回避バイアス（「1日を失っている」の暗示）が効く。

### 3.2 サブヘッドライン

```
無音カット、字幕生成、サムネイル作成まで。
動画をアップロードするだけで全部終わる。
```
**理由**: 具体的な機能を列挙して「何ができるか」を即座に理解させる。「アップロードするだけ」で操作の簡単さを提示。

### 3.3 各セクションのコピー

#### Navigation
```
SakuEdit | 機能 | 料金 | [無料で始める]
```

#### Hero
```

     無音カット、字幕、サムネまで自動で。
CTA: 無料で試す

```

#### ソーシャルプルーフ
```

**注**: UX調査「Proof before persuasion」原則に基づき、機能説明より前に配置。具体的な数値（丸めない）が効果的。

```

  Premiere Pro を開く → 素材を読み込む → 無音区間を手動で探す

右（SakuEdit）:
  動画をアップロード → 自動処理（3分） → 微調整 → 書き出し
  所要時間: 10分
```
**注**: 現行の「8時間 / 3分」のテキスト数字だけではなく、ワークフローの具体的なステップを視覚的に対比する。

#### コア機能セクション
```

### 3.4 CTA文言

| 位置 | メインCTA | サブCTA |
|------|---------|--------|
| Navigation | 無料で試す | — |
| Hero | 無料で試す | — |
| 料金 | 無料で試す | — |
| 最終CTA | 90分無料で試す | — |

**注**: 全てのCTAは `/home`（アップロード画面）へ遷移。

## 4. UIデザイン仕様

### 4.1 色設計（4分類）

#### ブランド色
| 変数名 | 色コード | 用途 |
|-------|---------|------|
| `--primary` | `#FF6B35` | ブランドアイデンティティ、CTAボタン |
| `--primary-dark` | `#E55A2B` | CTAホバー状態 |
| `--primary-light` | `#FF8F66` | アクセント背景（薄い使用） |

**注**: 現行の `#f97415` から `#FF6B35` に変更。より赤みのあるオレンジで、白背景との対比が強くなる。primary をCTAボタン**専用**とし、装飾には使わない。

#### 操作色
| 変数名 | 色コード | 用途 |
|-------|---------|------|
| `--primary`（上記と同一） | `#FF6B35` | メインボタン、リンク |
| `--secondary` | `#2D3748` | セカンダリボタン背景、ナビテキスト |

**原則**: ブランド色 = 操作色。ユーザーが `#FF6B35` を見たら「押せる」と理解する。

#### 状態色
| 変数名 | 色コード | 用途 |
|-------|---------|------|
| `--success` | `#10B981` | 完了、成功状態 |
| `--error` | `#EF4444` | エラー、失敗 |
| `--warning` | `#F59E0B` | 注意 |
| `--info` | `#3B82F6` | 補足情報 |

**原則**: 状態色はLP上では最小限の使用。デモ内の「完了」表示程度。

#### 中立色（背景・テキスト・ボーダー）
| 変数名 | 色コード | 用途 |
|-------|---------|------|
| `--background` | `#FFFFFF` | メイン背景 |
| `--background-alt` | `#F8FAFC` | セクション交互背景 |
| `--foreground` | `#1A202C` | メインテキスト |
| `--muted` | `#718096` | サブテキスト |
| `--border` | `#E2E8F0` | ボーダー、区切り線 |

**注**: 現行の暖色系背景（`#fcfaf8`）からニュートラルな白系に変更。暖色はブランド色のオレンジと干渉する。背景は中立にして、オレンジを際立たせる。

### 4.3 アニメーション方針

#### 原則
1. **初回表示のコンテンツにアニメーションを使わない** — ナビ、Hero、CTAは即座に表示
2. **スクロール連動アニメーションは最小限** — セクション単位での fade-in のみ許可
3. **意味のあるアニメーションのみ使用** — 状態変化（ホバー、デモの進行）に限定

#### 具体的な適用

| セクション | アニメーション | 理由 |
|----------|-------------|------|
| Navigation | なし | 即座に表示。ロゴのスライドインは不要 |
| Hero | なし | 最初のコンテンツは即座に全て見せる |
| コア機能 | セクション全体の fade-in（1回、0.3秒） | スクロールで自然に現れる程度 |
| デモ | ProcessingToEditorDemo の内部アニメーションのみ | デモ自体が動く要素。外枠のScrollRevealは不要 |
| 付加機能 | セクション全体の fade-in（1回、0.3秒） | コア機能と同様 |
| 料金 | なし | 情報を即座に読めることが重要 |
| CTA | なし | CTAは常に見えているべき |

#### 廃止するアニメーション
- `ScrollReveal` の direction（left/right）— 横からスライドするのは不要
- `StaggerContainer` / `StaggerItem` — 順番に表示する必要なし
- `Floating` — 浮遊アニメーションはLP不要
- `PulseRing` — LP不要
- `GradientText` — AIっぽさの原因
- `Parallax` — LP不要
- ロゴの `ScrollReveal direction="left"` — 即座に表示

#### 残すアニメーション
- `ProcessingToEditorDemo` 内部のプログレスアニメーション（改良の上で）
- CTAボタンのホバーエフェクト（`transition-colors`のみ）
- シンプルなセクション fade-in（`opacity: 0 → 1`、`y: 10px → 0`、duration: 0.3s）

#### Navigation

</nav>

## 5. 差別化ポイントの訴求

### 5.1 「サムネイルまで一気に作れる」の訴求方法

#### 位置づけ
コア機能の3つ目として、無音カット・字幕生成と同列に配置する。
競合調査の結果、**主要7サービスのいずれもLPでサムネイル生成を明示的に訴求していない**。これはSakuEditの明確な差別化ポイント。

#### コピー
```
サムネイルまで自動で作れる

テンプレートを選ぶだけ。動画フレームからも生成可能。
別ツールはもういらない。
```

#### 訴求のポイント
1. **「まで」の一言が重要**: 「編集**まで**」ではなく「サムネイル**まで**」。動画編集ツールの範囲を超えていることを示す
2. **別ツール不要の安心感**: Canva/Photoshopとの往復が不要
3. **統合ワークフロー**: 「同じ画面で」完結する価値

### 5.2 「高度な編集不要」ターゲットへのメッセージング

#### 明示的に言わない
- 「初心者でも使える」— 初心者呼ばわりは好感度を下げる
- 「プロ向け」— ターゲットに合わない
- 「簡単」— 曖昧で信頼性がない

#### 暗示的に伝える
- 「アップロードするだけ」— 操作の簡潔さを示す
- 「自動で」— 人間がやることの少なさを示す
- 「ワンクリックで」— 操作回数の少なさを示す

### 5.3 競合との差別化ポイント

| 差別化軸 | SakuEdit | 主要競合 |
|---------|----------|---------|
| **サムネイル同時生成** | 編集画面内で完結 | LP上で訴求なし（全7サービス） |
| **日本語ネイティブ** | 日本語UI・日本語ASR最適化 | Descript/Opus Clip/Pictoryは英語中心 |
| **ワンストップ** | 無音カット+字幕+サムネ+チャット編集 | 各社は特定機能に特化する傾向 |
| **操作の最小化** | アップロード→自動処理→微調整→書き出し | 多くの競合は編集UI操作が必要 |
