## 1. 現状分析

### 1.1 ユーザーのサムネイル作成における手間

| 作業 | 所要時間 | 痛み度 |
|------|---------|--------|
| ベース画像選定 | 15-30分 | 😖 |
| テキスト入力・配置 | 20-40分 | 😞😞 |
| 色味・デザイン調整 | 30-60分 | 😞😞😞 |
| フォント選び | 10-20分 | 😖 |
| A/Bテスト（複数パターン作成） | 60-120分 | 😞😞😞😞 |
| **合計** | **2-4時間** | |

**ポイント**: 動画編集後、さらに2-4時間の追加作業が必要

### 1.2 既存ツールの問題点

| ツール種別 | 問題点 |
|----------|--------|
| 専門デザインツール（Photoshop等） | 学習コスト高、オーバースペック |
| オンラインサムネ作成ツール | 素材の持ち出し・アップロードが面倒 |
| テンプレートサービス | カスタマイズの自由度が低い |

## 2. 価値チェーンの設計

### 2.1 従来のワークフロー

```
動画撮影 → 動画編集(数時間) → サムネイル作成(2-4時間) → アップロード
                                           ↑
                                    別ツールで作業
```

### 2.2 SakuEditのワークフロー

```
動画撮影 → SakuEditで動画編集 + サムネ生成（同画面） → アップロード
```

### 2.3 差別化ポイント

| 項目 | 従来ツール | SakuEdit |
|------|----------|----------|
| 動画編集 | 別ツール | 同一ツール |
| サムネ作成 | 別ツール | 同一ツール |
| 素材持ち出し | 必要 | 不要 |
| 画面切り替え | 必要 | 不要 |
| ワークフロー | バラバラ | 一貫性 |

**核となるメッセージ**: 「動画編集とサムネ作成が**同じ画面**で完結」

## 3. ターゲット訴求方法

### 3.1 「高度な編集がいらない人」へのアプローチ

| 訴求文 | ターゲットの心の声 |
|--------|-------------------|
| 「動画編集終わったと思ったら、サムネも作らなきゃ…」 | その通り |
| 「サムネ作るだけでも2時間かかるけど、AIなら？」 | やってみたい |
| 「別のツールを開くのは面倒だと思ったことは？」 | 毎回そう |
| 「デザインセンスなくても大丈夫です」 | 助かる |

### 3.2 メリットの3段階提示

```
【第1段階】時間削減
  サムネイル作成 2-4時間 → 30秒

【第2段階】ツール統合
  動画編集・サムネ作成が1つの画面で完結

【第3段階】デザインセンス不要
  プロ仕様のテンプレートから選ぶだけ
```

### 3.3 痛み点 → 解決策

| 痛み点 | 解決策 |
|-------|-------|
| 「サムネ作るの面倒」 | テンプレート選ぶだけ |
| 「別ツール使いたくない」 | 動画編集と同じ画面 |
| 「デザインできない」 | プロ仕様テンプレート |
| 「素材選びに時間がかかる」 | 動画フレームからそのまま |

## まとめ

ユーザーに伝えるべき核は「**動画編集とサムネ作成が同じ画面で完結し、別ツールが不要**」という点です。

時間削減（2-4時間 → 30秒）と統合性（1つの画面で完結）の2つの観点から訴求します。
