## 1. 新コンセプト定義

### 1.1 メインターゲット（再定義）

| ユーザータイプ | 特徴 | ニーズ |
|--------------|------|--------|
| **高度な編集がいらない人** | Premiere ProやDaVinci Resolveは使いこなしたくない | シンプルな操作で十分な品質の動画が欲しい |
| **動画編集を面倒に感じる人** | カット作業、テロップ合わせに時間を浪費している | 面倒な作業は自動化してほしい |
| **早く完成させたい人** | YouTube/TikTokの投稿ペースが速い | 作業時間を短縮したい |
| **編集スキルに自信がない人** | 自分でやると「素人っぽい」となる | プロっぽい見た目になりたい |

### 1.2 価値提案の軸転換

**旧（現行）**:
> YouTubeスタイルをコピーして、憧れのクリエイターみたいな動画を作る

**新（提案）**:
> AIがやるべきことをやるから、あなたは創作に集中できる

### 1.3 コアメッセージ

**「編集作業から解放され、コンテンツづくりに集中する」**

### 1.4 「YouTubeスタイルコピー」からどう移行するか

| 旧表現 | 問題点 | 新表現 | 効果 |
|-------|-------|-------|------|
| 「YouTubeスタイルを学習・再現」 | 競合との差別化が弱い | 「プロ仕様の編集を自動適用」 | 具体的な品質をイメージ |
| 「憧れのクリエイター」 | ニッチ感が強い | 「クリエイティブに集中できる」 | 幅広い層に訴求 |
| 「スタイルをコピー」 | 模倣というニュアンス | 「編集を楽にする」 | 解決感を強調 |

## 2. 情報優先順位設計

### 2.1 画面内の主役

**「AIが自動でやること」の具体性**

抽象的な「便利」ではなく、具体的な成果を見せる：
- 無音カット → 「無音・フィラーを自動カット」
- 字幕生成 → 「字幕トラック自動生成」
- サムネ生成 → 「サムネイルまで自動生成」

### 2.2 視線の流れ

```
① Hero: 痛み点 → 解決策（3秒で理解）
   └─ 「1日かかる作業を3分で終わらせる」

② Before/After: 具体的な成果を見せる
   └─ 10分動画 → 7分動画

③ 機能: 何が自動化されるか
   └─ 無音カット・字幕生成・サムネ作成

④ 価格: お試しのハードルを下げる
   └─ 90分無料体験

⑤ CTA: 今すぐアクション
   └─ 「無料で試す」
```

### 2.3 セクション構成

| セクション | 目的 | 要素 | 優先度 |
|----------|------|------|--------|
| Hero | 3秒で伝える | ヘッドライン + Before/Afterビジュアル | 最高 |
| 痛み点 | 共感を得る | 「編集時間」「面倒くささ」の具体例 | 高 |
| 解決策 | AIが何をするか | 無音カット・字幕・サムネ自動生成 | 高 |
| 体験 | 信頼感 | デモ/事例 | 中 |
| プラン | 選択の容易さ | Free体験 → 有料へのパス | 中 |
| CTA | アクション促進 | ハードルの低い開始ボタン | 高 |

## 3. コピー戦略

### 3.1 ヘッドライン案（5パターン）

| パターン | ヘッドライン | 特徴 | 推奨度 |
|---------|-------------|------|--------|
| A | 動画編集、AIにまかせて済ませませんか？ | 疑問形で思考を促す | ⭐⭐ |
| B | 10分の動画編集、1日かけてませんか？ | 具体的な数字で刺さる | ⭐⭐⭐ |
| C | 編集作業から解放される。コンテンツに集中する。 | 自由を売る | ⭐⭐⭐ |
| D | アップロードするだけ。あとはAIが。 | 最短経路を提示 | ⭐⭐⭐⭐ |
| E | **1日かかる作業を3分で終わらせる** | 数字の対比でインパクト | ⭐⭐⭐⭐⭐ |

**推奨**: パターンE（数値対比は最も理解しやすい）

### 3.2 サブヘッドライン案

| パターン | 文案 | 特徴 |
|---------|------|------|
| A | 無音カット・字幕生成・サムネ作成まで、全部自動 | 具体的機能を列挙 |
| B | 創作に集中してください。単純作業はAIに | 役割分担を明確化 |
| C | 編集スキル不要。動画をアップロードするだけ | ハードルを下げる |

### 3.3 CTAボタン文言

| 位置 | 文案 | 理由 |
|-----|------|------|
| メインCTA | 無料で試す | 「登録」よりハードル低い |
| サブCTA | デモを見る | 興味を深める |
| 下部CTA | 90分無料体験 | 具体的な数字で安心感 |

### 3.4 AIっぽさを消す表現

| 旧表現（避ける） | 新表現（推奨） |
|-----------------|--------------|
| × 「AIが自動で〜」 | ○ 「自動で〜」または「〜が自動で行われます」 |
| × 「AI分析で〜」 | ○ 「分析で〜」 |
| × 「AI搭載の〜」 | ○ 「高度な機能を備えた〜」 |

**理由**: 「AI」という単語は既に意味が薄まりすぎている。「何ができるか」を伝える方が価値が伝わる。

## 4. 色設計ガイドライン

### 4.1 推奨配色

| 色種類 | 色コード | 役割 | 理由 |
|-------|---------|------|------|
| **ブランド色** | `#FF6B35` | アクション・ブランドアイデンティティ | 暖色系で創造性・情熱を表現。視認性高い |
| **操作色（プライマリ）** | `#FF6B35` | メインボタン・重要アクション | ブランド色と統一。目を引く |
| **操作色（セカンダリ）** | `#2D3748` | サブボタン・ナビゲーション | 中立色で干渉しない |
| **成功色** | `#10B981` | 完了・成功状態 | 緑は universally ポジティブ |
| **エラー色** | `#EF4444` | エラー・警告 | 赤は universally ネガティブ |
| **情報色** | `#3B82F6` | ヒント・補足情報 | 青は冷静・信頼性 |
| **背景（メイン）** | `#FFFFFF` | メイン背景 | クリーンで読みやすい |
| **背景（セクション）** | `#F7F9FC` | セクション区分 | 目に優しい薄いグレー |

### 4.2 色の原則

1. **同じ色に複数の意味を持たせない**
   - ブランド色はアクションのみに使用
   - 成功/エラー/情報は直感的な色（緑/赤/青）

2. **ブランド色 = 操作色**
   - ユーザーがブランド色と同じ色のボタンをクリックしたら、期待通り動く

3. **直感的な色を採用**
   - 成功 = 緑
   - エラー = 赤
   - 情報 = 青
   - 警告 = 黄（必要に応じて）

### 4.3 既存のグローバル変数との整合性

現在のプロジェクトで使用されている変数を確認し、必要に応じて以下のマッピングを提案：

| 変数名 | 提案する色 | 割り当て役割 |
|-------|----------|------------|
| `--color-primary` | `#FF6B35` | ブランド色・メイン操作色 |
| `--color-secondary` | `#2D3748` | サブ操作色・テキスト |
| `--color-success` | `#10B981` | 成功状態 |
| `--color-error` | `#EF4444` | エラー状態 |
| `--color-info` | `#3B82F6` | 情報・ヒント |

## 5. サムネイル機能の売り方

### 5.1 価値チェーン

**従来のワークフロー**:
```
動画撮影 → 動画編集(数時間) → サムネイル作成(2-4時間) → アップロード
                                           ↑
                                    別ツールで作業
```

**SakuEditのワークフロー**:
```
動画撮影 → SakuEditで動画編集 + サムネ生成（同画面） → アップロード
```

**核となるメッセージ**: 「動画編集とサムネ作成が**同じ画面**で完結」

### 5.2 「サムネまで」の伝え方

| 伝え方 | 文案 |
|-------|------|
| 機能として | 「字幕だけでなくサムネまで自動生成」 |
| コスト削減として | 「別途サムネ作成ツールはいりません」 |
| 統合感として | 「動画編集と同じ画面でサムネも作れる」 |

### 5.3 時間対比

| 手動 | AI |
|-----|----|
| 2-4時間 | 30秒 |

### 5.4 機能セクションでの伝え方

```
Feature 03: サムネイルまで自動生成

テンプレート選ぶだけ。
動画フレームからそのまま生成も。

[テンプレート] [素材UP] [動画から]

最大4枚同時生成 → 気に入ったのを採用

別ツールはもういりません
```

### 5.5 料金表での表現

| 機能 | Free | Pro | Business | Enterprise |
|------|------|-----|----------|------------|
| **サムネ生成** | - | **10枚/月** | **50枚/月** | 無制限 |

## 6. 実運用想定

### 6.1 長いテキスト対策

- セクションごとに最大3行（約40文字）に制限
- 「もっと見る」で展開するUIパターン準備
- 行間・文字サイズで読みやすさ担保

### 6.2 欠損データ対策

| データ種別 | 欠損時の対処 |
|----------|------------|
| 事例画像 | スケルトンローダー + キャッチコピーで代替 |
| 数値データ | 「〇〇人中〇〇人が満足」ではなく「〇〇万人以上が利用」で概算可 |
| テストimonials | 事例が揃うまで「〜」で代替 |

### 6.3 ゼロ件対策

| 画面 | ゼロ件時の表示 |
|------|--------------|
| プロジェクト一覧 | 「最初のプロジェクトを作りましょう」CTA |
| テンプレート一覧 | 「お好みのスタイルを選択してください」文言 |
| お気に入り | 「お気に入りはまだありません」 |

### 6.4 UI崩れ対策

- Flexbox/Gridの自動レイアウト崩れを防ぐ
- 最大幅・最小幅を指定
- 画像のaspect-ratioを固定
- テキストの文字数制限（line-clamp）
