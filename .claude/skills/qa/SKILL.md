# QA Checklist

環境検証を最初に行い、ブロッカーを特定してから画面テストに進む。

## Step 1: 環境健全性チェック
1. `.env` の全必須変数が設定されているか確認
2. 外部API（OpenRouter, ZAI等）への接続テスト
3. AIモデルが非推奨でないか確認
4. DBにシードデータが存在するか確認
5. `npm run build` が成功するか確認

## Step 2: 各画面の体系的テスト
各画面について以下を実施:
- ユーザーフロー（作成、編集、保存、リロード、永続性確認）
- デスクトップとモバイル両方のレンダリング確認
- エラー状態のハンドリング確認

## Step 3: 結果レポート
以下の形式でまとめる:
- **動作OK**: 正常に動作する機能
- **環境ブロック**: 環境設定の問題でブロックされた機能
- **実バグ**: コードの問題によるバグ

各バグについて: ファイルパス、期待動作 vs 実際の動作、重要度を記載。
